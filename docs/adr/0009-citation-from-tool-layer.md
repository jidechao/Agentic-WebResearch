# ADR-0009: 引用事实源锚定在搜索工具层（而非 LLM 自报）

- 状态：已接受
- 日期：2026-07-20

## 背景

最终报告当前全部是自然语言文本——"来源"只靠模型在子任务总结时自觉保留（SUBTASK_SYSTEM 里"保留关键数据、时间和来源"是软要求）。用户无法从报告某条断言跳到支撑它的证据。模型可能丢链接、改写、凭记忆编造。可信度无法验证，溯源成本转嫁给读者。

要为报告增加引用（Citation），核心问题是：**URL 与标题由谁说了算？**

## 决策

### 引用的事实源锚定在联网搜索工具层

证据 ID（s1/s2/...）与结构化记录（{id, title, url, date}）由 web_search 工具在解析博查 API 返回时分配并写入全局证据池。**参考列表条目的 URL/title 由证据池（API 直取）唯一决定；报告后处理只按 [sN] 查池展开，模型无法向参考列表注入自编 URL。**（模型在 ReAct 消息里仍会看到工具返回的 URL 文本，但它对参考列表的最终 URL 无影响——后者由池唯一决定。）

报告正文引用用运行期 ID [sN]；报告生成后由后处理重映射为连续展示编号 [1]..[n]，并拼接参考列表。见 docs/specs/citation.md。

### 为什么不交给 LLM 自报

| 方案 | 否决/采纳理由 |
|---|---|
| **工具层锚定（采纳）** | URL 来自 API，零幻觉；ID 全局唯一，并发安全；模型只复制它见过的 [sN] |
| LLM 在子任务总结时输出 citations | URL 不保证真实——模型可能凭记忆编、漏、或把搜过但没用到的也列上 |
| 报告生成时让模型补 citations | 最晚最不可靠——原始搜索结果早已是模糊自然语言，模型只能猜 |

### 配套决策（经七问逐一确认）

| 维度 | 选择 |
|---|---|
| 形态 | 正文内联 [N] + 末尾参考列表 |
| 作用域 | 全 run 全局池，itertools.count 原子分 ID |
| 展示编号 | 报告期重映射成连续 [1]..[n]（纯展示，模型见的 [sN] 运行期 ID 不变） |
| 幻影引用 | 删除（池里没有的 [sN] 不进参考列表，不触发重写） |
| 字段 | {id, title, url, date} + URL 规范化去重（去 fragment + utm_ + 小写 host，保留首次 ID） |
| 持久化 | runs/<run_id>/evidence.json，/resume 回填，counter 起点 len+1 |

## 为什么是全局池而非每子任务一个池

每子任务一个池会导致 ID 跨子任务碰撞（每个池都从 s1 起，报告合并时 [s1] 指向不明，无法从文本消歧）。全局池从源头消灭碰撞。代价是编号不连续——但连续性由报告期重映射解决，运行期 ID 唯一即可。两层各管各的：运行期 ID 保证模型不编号错乱，展示 ID 保证阅读美观。

## 为什么不改 Critic / report_review 的 schema

现有"证据充分性"维度已间接约束引用质量。给 Critic 加 citation 维度会让 schema 膨胀，且 Critic 看到的是子任务 result（自然语言），无法验证 [sN] 是否真实对应池中条目——验证逻辑只在后处理这一处。若实测发现模型普遍不标引用，再行评估。

**执行时机呼应**：finalize_report 在反思循环结束后、落盘前执行一次。因此 report_review 看到的是原始 [sN] 标记而非重映射后的 [k]——"证据充分性"维度可直接评判模型是否标注了引用，标得差即触发重写，循环天然纠偏。

## 并发安全

DAG 引擎多子任务并发调 web_search（asyncio.to_thread 包装同步 ReAct 循环）。ID 分配的计数器递增：itertools.count.__next__ 在 CPython GIL 下原子。但 URL 去重是"查表未命中 → 分配新 ID → 写入"的 check-then-set 序列，不在单条字节码内——并发下两个子任务同搜一个 URL 可能各自分到不同 ID、致去重失效。因此 add_evidence 用一把模块级互斥锁包住整个查询-分配-写入序列。迁移到自由线程构建（PEP 703）或多进程执行时需复核锁范围。

## 未采纳项（防重提）

- **LLM 自报 citations**：URL 幻觉风险，否决（理由见上表）。
- **每子任务一个池**：ID 碰撞，否决。
- **编号不重映射（保留 [sN] 终态）**：纯审美取舍，连续性更好且成本仅约 15 行后处理，采纳重映射。
- **保留幻影引用并标注"未验证"**：假锚点不如裸断言诚实，否决。
- **存 summary 进池**：参考列表不展示正文摘要，纯冗余，否决。
- **把 Citation 评审加进 Critic / report_review schema**：schema 膨胀且 Critic 无法验证 ID 真实性，否决。
- **无锁去重（接受极低概率重复）**：成本更低，但无法兑现用户故事 #8"同 URL 只算一条"的硬保证；采纳加锁（约 3 行）换取保证成立。

- 离线：新增测试文件——URL 规范化去重、ID 全局递增与 reset、带历史 reset 起点、后处理重映射连续性、幻影删除、未引用条目不进列表。
